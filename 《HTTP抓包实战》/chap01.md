# 读书笔记

- HTTP报文分为响应报文和请求报文

- Fiddler 本质上是一个Web 代理服务器。它的默认工作端口是8888

- 默认端口是8888

- 本质上是一个web代理服务器

- 代理服务器的优点

  - 共享网络，可以让局域网用户同时上网
  - 提高访问速度，因为有缓存
  - 突破访问限制，当访问权限受到限制时，可以使用有权限的代理服务器。
  - 隐藏身份，使用代理服务器做反向代理

- Fiddler使用的代理地址是127.0.0.1:8888，所以在启动Fiddler的时候，会把Internet选项中的代理修改为127.0.0.1:8888，然后退出的时候会取消修改。

  <!-- 如果是非正常退出，则可能造成网络无法访问，这时可以重启fiddler -->

- 达到Web安全的两个因素

  - 浏览器和Web服务器之间的内容只有浏览器和Web服务器可以看到通信的具体内容
  - Http请求内容和请求响应不会被第三方篡改

- 如果Web服务器对所有客户端使用同样的对称加密算法，那就相当于没有加密。因此应该对不同的客户端使用不同的对称加密算法。这就需要协商。

- 状态码200表示成功响应

- 状态码204表示没有内容，只有一个Header和一个状态行

  - 在不获取资源的情况下了解资源的情况
  - 通过查看HTTP响应中的状态码查看某个对象是否存在
  - 通过查看Header测试资源是否被修改

- 状态码206是部分内容

  - 一些下载工具就是通过206状态码来实现端点续传的。
  - 将一个大文档分解为多个下载段同时下载，比如在线看视频。
